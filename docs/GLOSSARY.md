# Glossary

| Term | Definition |
|------|------------|
| **Alchemy** | RPC provider for reading blockchain state across 7 EVM networks |
| **AppKit** | WalletConnect's SDK for connecting wallets to web apps |
| **Chain** | A blockchain network (Ethereum, Polygon, Arbitrum, etc.) |
| **CSR** | Client-Side Rendering — rendering in the browser via JavaScript |
| **Edge Function** | Supabase's serverless function runtime (Deno) for backend logic |
| **EVM** | Ethereum Virtual Machine — compatible chains (Ethereum, Polygon, etc.) |
| **ISR** | Incremental Static Regeneration — Next.js pattern for updating static pages |
| **JWT** | JSON Web Token — used for Supabase session management |
| **KYC/AML** | Know Your Customer / Anti-Money Laundering — compliance requirements |
| **Non-custodial** | User retains full control of their private keys; platform never holds funds |
| **OpenCode** | AI coding assistant that reads documentation to implement features |
| **PITR** | Point-in-Time Recovery — database backup/restore to any second |
| **Receiving Address** | Platform's configured wallet address for crypto payment collection |
| **RLS** | Row-Level Security — PostgreSQL policies that restrict row access per user |
| **RPC** | Remote Procedure Call — blockchain node communication protocol |
| **RSC** | React Server Component — renders on the server, sends HTML to client |
| **Server Action** | Next.js pattern for server-side form mutations called from client |
| **SSR** | Server-Side Rendering — rendering pages on the server per request |
| **viem** | TypeScript-first Ethereum library for reading chain state and verifying txns |
| **wagmi** | React Hooks for Ethereum — wallet connection, balances, chain state |
| **WalletConnect** | Open protocol for connecting wallets to dApps via QR code / deep link |
