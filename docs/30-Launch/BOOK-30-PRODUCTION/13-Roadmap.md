# Post-Launch Roadmap

## Phase 1: Launch (v1.0.0)

```text
Status: Current
Focus: Core platform stability

Features:
- Wallet authentication (WalletConnect v2, MetaMask, Coinbase, Trust Wallet)
- Crypto-to-fiat card funding
- Payment verification with on-chain confirmation
- Card management dashboard
- Transaction history
- Admin portal
- RBAC (customer, admin, super_admin)
- Audit logging
- 28 UI components
- CI/CD pipeline
- Database schema (19 tables, 65 indexes)
```

## Phase 2: Growth (v1.1.0 — v1.3.0)

### v1.1.0 — Referral Program

```text
Features:
- Referral link generation
- Referral tracking dashboard
- Rewards system
- Referral analytics
- Estimated: 6 weeks post-launch
```

### v1.2.0 — Advanced Analytics

```text
Features:
```

### v1.3.0 — Partner & Affiliate Portal

```text
Features:

- Partner analytics
- Estimated: 14 weeks post-launch
```

## Phase 3: Scale (v2.0.0 — v3.0.0)

### v2.0.0 — Enterprise APIs

```text
Features:
- Public REST API
- API keys and rate limiting
- Webhook management
- Developer portal
- API documentation
- SDK generation
- Estimated: 6 months post-launch
```

### v3.0.0 — Global Expansion

```text
Features:
- Multi-region deployment
- Multi-language support (i18n)
- Localized card products
- Regional compliance (EU, UK, Asia)
- Advanced KYC/AML integration
- Fiat on-ramp (if regulatory requirements met)
- Estimated: 12+ months post-launch
```

## Roadmap Governance

| Rule | Description |
|------|-------------|
| Priorities may shift | Based on user feedback, market conditions, regulatory changes |
| No scope creep | Each release has a defined scope; features not making the cut go to next release |
| Regular review | Roadmap reviewed quarterly by CTO + Product |
| Transparency | Roadmap published on status page (future) |
